# GenBlox

Минимальный прототип браузерной платформы игровых спейсов. Сейчас доступны
крестики-нолики 3 × 3 и русские шашки.

## Как играть

### Без комнаты

Нажмите **«Играть одному»** — запустится локальная партия против компьютера.

### Открытая комната

1. Откройте меню **«Комнаты»** в шапке.
2. Нажмите **«Найти открытую комнату»**.
3. Playroom подключит вас к ожидающей комнате. Если свободных комнат нет,
   автоматически будет создана новая.

Второй игрок повторяет те же действия и попадает в уже созданную свободную
комнату. После подключения любой участник может нажать **«Запустить для всей
комнаты»**. Игра откроется у всех.

### Приватная игра

1. Откройте меню **«Комнаты»** и нажмите **«Создать приватную комнату»**.
2. Передайте другу код, ссылку или QR из меню комнаты.
3. Друг открывает ссылку либо вводит код в том же меню.

Комната является общим хабом, а не одной партией. Закрытие игры возвращает всех
участников в каталог без выхода из комнаты.

Для выхода откройте меню комнаты и нажмите **«Выйти из комнаты»**. Страница
перезагрузится без кода комнаты, завершая Playroom-сессию.

PlayroomKit создаёт временную комнату и синхронизирует состояние. Supabase,
собственный сервер и база данных не нужны.

## Локальный запуск

Нужен любой статический HTTP-сервер:

```bash
npx serve .
```

Открывать `index.html` напрямую через `file://` не рекомендуется.

## Архитектура

```text
index.html                       разметка экрана и игры
styles.css                       весь визуальный слой
js/app.js                        контроллер каталога, комнаты и активной игры
js/core/playroom-client.js       единственная граница с PlayroomKit
js/creator/room-delivery.js      доставка и проверка TXT внутри комнаты
js/creator/room-runtime.js       sandbox-запуск пользовательской игры у участников
js/games/tic-tac-toe.js          правила крестиков-ноликов
js/games/checkers.js             правила русских шашек
```

Правила игры не обращаются к DOM и сети. Комната хранит `activeGame`, поэтому
один участник может запустить спейс сразу у всех. UI не вызывает PlayroomKit
напрямую: интеграция спрятана в адаптере.

## Развёртывание

Для production-развёртывания используется `pnpm build`, а публикуемой директорией
является `dist`. PlayroomKit и QR загружаются с CDN. Creator использует Supabase URL и
публичный anon key из серверных переменных окружения; секретные ключи в клиент не попадают.

Без сборки основной каталог и локальный Creator продолжают работать через обычный
статический HTTP-сервер, но `Save online` будет показан как ненастроенный.
