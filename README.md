# GenBlox

Браузерная платформа маленьких игровых пространств. Прототип включает
пиксельное меню и два спейса для двух игроков:

- классические крестики-нолики 3×3;
- «5 в ряд» на поле 10×10.
- «Морской бой» на поле 8×8 против компьютера или другого игрока.

## Локальный запуск

Проект не требует сборки или установки зависимостей:

```powershell
python -m http.server 3000
```

Затем откройте `http://localhost:3000`.

Для публикации на Vercel достаточно импортировать репозиторий как статический сайт.

## Файлы приложения

- `index.html` — каркас меню и игровых окон.
- `styles.css` — пиксельный интерфейс и адаптивная вёрстка.
- `js/app.js` — координация интерфейса, комнат и активной игры.
- `js/games/catalog.js` — каталог спейсов и общие правила игр в ряд.
- `js/games/battleship.js` — состояние и правила морского боя.

## Сетевая игра

1. На первом устройстве нажать «Играть» → «Создать комнату».
2. Передать появившийся шестизначный код второму игроку.
3. На втором устройстве нажать «Играть», ввести код и выбрать «Войти».

Соединение между браузерами работает через Supabase Realtime. Таблицы базы
данных не используются: Presence формирует каталог комнат, а Broadcast
передаёт запросы на вход, ходы и состояние матча.

Доступны два типа комнат:

- открытая — видна всем игрокам в списке;
- приватная — доступна только по шестизначному коду.

Перед входом второго игрока владелец комнаты принимает или отклоняет запрос.

### Настройка Supabase

1. В Supabase Dashboard открыть **Project Settings → API**.
2. Скопировать Project URL и Publishable key. Для старого проекта допустим
   legacy `anon` key.
3. Вставить значения в `config.js`:

```js
window.GENBLOX_CONFIG = {
  supabaseUrl: "https://PROJECT.supabase.co",
  supabaseKey: "sb_publishable_..."
};
```

`service_role`, secret key, JWT secret и пароль базы нельзя помещать в
`config.js` или любой другой файл клиентского приложения.

Если сайт развёрнут на Vercel, достаточно добавить там переменные
`SUPABASE_URL` и `SUPABASE_PUBLISHABLE_KEY` либо `SUPABASE_ANON_KEY`.
Функция `api/config.js` безопасно передаст браузеру только эти публичные
значения. Остальные переменные из `.env` она не читает.

### Развёртывание

После заполнения `config.js` сайт разворачивается на Vercel как обычный
статический проект. Отдельный backend-деплой не требуется.

## Процесс разработки

Документы процесса разработки:

- [План-концепт](PLAN.md) — цели, границы, этапы и критерии готовности проекта.
- [Канбан](KANBAN.md) — текущее состояние задач и правила движения карточек.

## Как работать с документами

1. Перед началом этапа уточнить соответствующий раздел в `PLAN.md`.
2. Добавить конкретные задачи этапа в `KANBAN.md`.
3. Во время работы перемещать задачи между колонками канбана.
4. После завершения этапа обновить прогресс и зафиксировать принятые решения.
