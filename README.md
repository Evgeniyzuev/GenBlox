# GenBlox

Минимальный прототип браузерной платформы игровых спейсов. Сейчас в проекте один
спейс: сетевые крестики-нолики 3 × 3 для двух игроков.

## Как играть

1. Откройте сайт и нажмите **«Играть онлайн»**.
2. В лобби Playroom задайте имя и поделитесь с другом кодом или QR.
3. Хозяин нажимает **Launch**, когда подключились два игрока.

PlayroomKit создаёт временную комнату и синхронизирует состояние. Supabase,
собственный сервер и база данных не нужны.

## Локальный запуск

Нужен любой статический HTTP-сервер:

```bash
npx serve .
```

Открывать `index.html` напрямую через `file://` не рекомендуется.

## Архитектура

```text
index.html                       разметка экрана и игры
styles.css                       весь визуальный слой
js/app.js                        сценарий UI
js/core/playroom-client.js       единственная граница с PlayroomKit
js/games/tic-tac-toe.js          чистые правила игры
```

Правила игры не обращаются к DOM и сети. UI не вызывает PlayroomKit напрямую:
вся интеграция спрятана в адаптере. Новый спейс должен добавляться отдельным
модулем в `js/games`, а не расширять правила существующей игры.

## Развёртывание

Проект статический и подходит для Vercel без build-команды. Токены и `.env` не
требуются. PlayroomKit загружается с официального CDN.
