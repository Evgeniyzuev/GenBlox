# GenBlox — project memory

Краткие долговременные решения. Текущие задачи и прогресс находятся в `KANBAN.md`,
подробная архитектура — в `PLAN.md`.

## Creator

- Основной переносимый формат игры — один текстовый файл `*.genblox.txt`, не ZIP и не PDF.
- Файл содержит секции `MANIFEST`, `HTML`, `CSS` и `JAVASCRIPT` со стабильными маркерами.
- Загрузка/вставка используют один parser; скачивание/копирование — один serializer.
- Большие встроенные игры отдают компактные самостоятельные ремикс-шаблоны, а не полный исходник.
- Весь пользовательский интерфейс и обучение сайта написаны простым английским.
- Пользовательский код выполняется только в sandboxed iframe со строгой CSP и без доступа к данным сайта.
- Игра взаимодействует с платформой только через версионируемый ограниченный SDK.
- До модерации серверная версия будет видна только автору и приглашённым друзьям.
- Встроенный AI-чат не входит в MVP: ребёнок использует любой внешний AI-чат.
- Публичная публикация невозможна до появления аккаунтов, версий, изоляции и модерации.
- Обучение ребёнка является частью MVP, а не необязательной полировкой.

## Текущий рубеж

Локальный MVP включает технический Clicker, Tic-Tac-Toe remix с ботом и Sky Runner с
Canvas, keyboard/touch-управлением. Creator умеет явно переключать шаблоны, импортировать
и экспортировать TXT, безопасно запускать preview, работать с SDK v1, обучать ребёнка и
хранить до 10 локальных версий. Tic-Tac-Toe проверяет полный AI-цикл для пошаговой игры,
а Runner — непрерывный игровой цикл. Следующий рубеж — проверка Runner через внешний AI;
аккаунты, приватное тестирование, модерация и мультиплеер следуют отдельными этапами.
